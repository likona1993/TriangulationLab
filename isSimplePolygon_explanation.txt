Как работает isSimplePolygon (src/core/geometry/ear_clipping.hpp)
====================================================================

Назначение: проверить, что никакие два НЕСМЕЖНЫХ ребра полигона не
пересекаются. Это необходимое условие для ear clipping — алгоритм
в принципе не определён для самопересекающихся контуров.

1. orientation(p, q, r, eps) — ориентация тройки точек
--------------------------------------------------------------
    val = (q.y - p.y) * (r.x - q.x) - (q.x - p.x) * (r.y - q.y)

Это знак векторного произведения (q-p) x (r-q) — стандартный тест
"поворота" при движении p -> q -> r:
  - val >  eps  -> поворот против часовой стрелки   -> return 1
  - val < -eps  -> поворот по часовой стрелке        -> return 2
  - |val| <= eps -> точки лежат на одной прямой       -> return 0

Тот же cross-product паттерн, что и в isConvex / removeCollinearPoints,
но здесь важны все три исхода (лево/право/на прямой), а не просто
"ноль или нет".

2. onSegment(p, q, r, eps) — лежит ли точка внутри отрезка
--------------------------------------------------------------
Вызывается ТОЛЬКО когда уже известно, что три точки коллинеарны
(orientation == 0). В этом случае "лежит на отрезке" эквивалентно
"лежит внутри его ограничивающего прямоугольника" — проверяется
через inRange отдельно по x и по y.

3. segmentsIntersect(p1, q1, p2, q2, eps) — пересекаются ли отрезки
--------------------------------------------------------------
Два независимых случая:

  a) Общий случай (строгое пересечение "крест-накрест"):

       if (o1 != o2 && o3 != o4) return true;

     o1/o2 — по разные ли стороны от прямой p1-q1 лежат p2 и q2.
     o3/o4 — по разные ли стороны от прямой p2-q2 лежат p1 и q1.
     Отрезки пересекаются тогда и только тогда, когда оба условия
     верны одновременно — концы одного отрезка лежат по разные
     стороны от другого, и наоборот.
     Пример: диагонали квадрата (0,0)-(1,1) и (0,1)-(1,0) пересекаются
     в центре — эту ситуацию ловит именно эта ветка.

  b) Коллинеарные особые случаи (когда одна из четырёх троек лежит
     на одной прямой, orientation == 0):

       if (o1 == 0 && onSegment(p1, p2, q1, eps)) return true;
       ... и ещё 3 симметричных проверки (o2/o3/o4)

     Покрывает:
       - T-образное касание — вершина одного ребра лежит точно на
         середине другого ребра (не пересечение диагоналей, а именно
         "упирается" в чужое ребро);
       - перекрытие коллинеарных отрезков — например, если два ребра
         полигона лежат на одной прямой и частично перекрываются.

     Без этих 4 проверок общий тест (o1 != o2 && o3 != o4) пропустил
     бы коллинеарные случаи, потому что там o1 == o2 == 0 (не "разные
     стороны").

4. isSimplePolygon(poly, eps) — перебор всех пар рёбер
--------------------------------------------------------------
    for i in 0..n:
        for j in i+1..n:
            if (j == i_next || j_next == i): continue   // смежные - пропуск
            if segmentsIntersect(edge_i, edge_j): return false
    return true

Перебираются все пары рёбер (i, i+1) и (j, j+1) с j > i — O(n^2)
проверок. Это приемлемо, поскольку сам ear clipping и так O(n^3)
в худшем случае (внешний цикл отрезания ушей x линейный поиск уха x
линейная проверка "пусто ли внутри треугольника").

Почему пропускаются смежные рёбра: соседние рёбра всегда имеют общую
вершину — например, ребро i заканчивается там же, где начинается
ребро i+1. Формально segmentsIntersect для них тоже вернёт true (общая
точка — это и есть "пересечение"), но это не самопересечение полигона,
а нормальная структура контура.
  - j == i_next    — ловит "соседние по порядку" рёбра;
  - j_next == i    — ловит ребро в конце массива и ребро в начале
                     (замыкание контура, wraparound).

5. Пример на самопересекающемся полигоне (bowtie) из теста
--------------------------------------------------------------
Полигон: {(0,0), (1,1), (0,1), (1,0)}

Рёбра e0=(0,0)-(1,1) и e2=(0,1)-(1,0) не смежные. Это диагонали
"бабочки", и segmentsIntersect для них сработает через общий случай:
(0,1) и (1,0) лежат по разные стороны от прямой (0,0)-(1,1), и
наоборот — значит есть пересечение.

isSimplePolygon вернёт false, и triangulate() сразу откажет с ошибкой
"Polygon is self-intersecting", не пытаясь резать уши.
